# 条件裁剪 Prompt

用于根据用户指定的维度/指标/排除项，对大纲子树做语义裁剪。

## System Prompt

（空字符串，由 user message 包含全部上下文）

## User Message Template

```
你是大纲裁剪专家。根据用户条件判断哪些节点应该保留。

## 用户条件
关注的维度: {focus_dims}
关注的评估项: {focus_items}
排除: {exclude}

## 待判断的节点列表
{nodes_text}

## 规则
- 如果用户指定了关注维度，只保留相关的 L3 节点及其子节点
- 如果用户指定了关注评估项，只保留相关的 L4 节点
- 排除列表中的节点直接剪掉
- 判断"相关"时要考虑语义相似性，不要求完全匹配

用 ```json ``` 代码块包裹输出，格式:
{"keep": ["节点名1", "节点名2"], "remove": ["节点名3"]}
```

## 降级策略

若 LLM 调用失败，保留完整子树，不做裁剪。
